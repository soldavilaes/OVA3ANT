  • DEFINICIÓN

    La pérdida del conocimiento es considerada un problema que ocurre en la evaluación recursiva de funciones estructurales como funciones de procesamiento de listas.

    Básicamente, está pérdida de conocimiento se ocasiona cuando una función específica crea una lista con el primer elemento, pero después la segunda instancia de esta función no tiene forma de saber cuál es el primer elemento, por lo cual el conocimiento se pierde.
    El problema de la pérdida de conocimiento se produce debido a que las funciones
    recursivas son independientes de su contexto.

    Solución a la Pérdida de Conocimiento

    Para resolver el problema de la pérdida de conocimiento se deben utilizar acumuladores.

Un ACUMULADOR es un parámetro que puede recibir una función, el cual guarda información de manera incremental de forma que su contenido actual depende de un valor precedente.

Frecuentemente los valores del acumulador se van generando por medio de sumas o restas sucesivas.

A continuación, se muestra la expresión genérica de un acumulador denominado A.

Suma sucesiva.

EJEMPLO

...Otra forma de realizar el ejemplo 1

El ejemplo de distancias anteriormente desarrollado, ahora se solucionará utilizando el parámetro adicional acum, el cual es un acumulador que resuelve la pérdida de conocimiento.

acum representa la distancia acumulada, ya que almacena el valor de las distancias absolutas, el cual se va actualizando durante el procesamiento de la función.

El parámetro acum se inicializa con el valor de cero.

A partir de esta definición se puede determinar que cuando se hace el llamado recursivo de la función, esta recibe como primer parámetro el resto de la lista y como segundo parámetro la distancia absoluta del punto actual, cuyo cambio depende del primer parámetro de la función rel-2-abs.

>

La evaluación de la función rel-2-abs para la lista (cons 3 (cons 2 (cons 7 empty))) es la siguiente:


(rel-abs (list 3 2 7) 0)   acum = 0

= (cons 3 (rel-abs (list 2 7) 3))   acum = 3
= (cons 3 (cons 5 (rel-2-abs (list 7) 5)))   acum = 5
= (cons 3 (cons 5 (cons 12 (rel-2-abs empty 12))))   acum = 12
= (cons 3 (cons 5 (cons 12 empty))); Lista de distancias absolutas

Las claves para el desarrollo de una función de estilo acumulador son:

1 .

Reconocer que la función se beneficia de, o necesita, un acumulador.

2 .

Entender qué representa el acumulador.

Despues de este analisis, se debe estudiar la función y buscar una de las siguientes características:

Si la función es estructuralmente recursiva y si el resultado de una aplicación recursiva es procesado por una función recursiva auxiliar, entonces se debe considerar el uso de un parámetro acumulador.

Si se trata de una función basada en recursión generativa, se debe comprender si el algoritmo no puede producir un resultado para las entradas para las cuales se espera un resultado. Si es así, puede ayudar agregar un parámetro que acumule conocimiento.
